Duxbury Document Style Guide

One of the single most important things to be aware of when authoring text, is to use Styles.

This is not really any different to using Styles in Word for Windows, which of course is highly recommended when using DBT.

However we do need to define a "Corporate set of Styles" for all documentation.

Fortunately, Flare's use of *.css files allows Styles to be defined for various types of output.

For now, we ideally need to re-define Heading and Normal Styles for XHTML and Printed output. Others will follow.
