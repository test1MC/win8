# embosser configurations using the embosser you are working with, and then create new ones
# using the name of the modified embosser. Otherwise, you run the risk of making changes
# in the myemb.elt textfile, but not seeing the effect of your changes in the next embossing test.
#
# The myemb.elt textfile is easy to modify, with a format that is close to
# self-documenting. If you make a serious error, you will get an error that
# says something like Syntax error in emb.elt at or near line 13. The error message
# may be misleading. The error is either line 13 of file myemb.elt or line 13 of file
# emb.elt. Since you are doing the changes, you know which file has the error.
# If you are following these instructions and not making any changes to emb.elt, any
# errors must be in myemb.elt. If you feel you cannot fix your problem, you can always
# revert back to your original setup by deleting myemb.elt.

# One reason to adjust this file is to deal with flow control (handshaking) problems. Please see the notes
# at the end of this file for more on this subject
#
#  Here are the properties, and what they mean:
#
#
#   AdditionalDelay     / milliseconds added to each character when OneByOne is used
#   Baud = 4800         / The default for the baudrate (usually 9600)
#   BreakLimits         / do not limit the carriage width or form length
#   CanDoInterpoint     / an interpoint embosser. A user can still set an
#                       / embosser for single sided
#                       / or ask for a job to be done single sided
#   ControlDelay        / delay for each control character when OneByOne is used
#   ControlD            / send a control-D at the end of a job (last copy only)
#  
#   ControlZ            / send a control-Z at the end of a job (last copy only)
#  
#   CrMac               / only send carriage return, not carriage return / line feed 
#   Customize           / a parameter having a special function for this embosser
#                       / Used by Nippon Gemini to control line spacing
#                       / Customize = 2  means small spacing,  3 is medium
#                       / 4 is large (the default), 6 means SP
#                       / Used by Index to control which parameter controls graphics mode (default is 27)
#                       /  and how many parameters total are in the initialization sequence (default is 40)
#                       /  Value is 256 * number of parameters + graphics resolution parameter index
#   DataBits = 7        / The default for databits (usually 8)
#   Driver = XXX        / which group of output programs to use
#
#   Driver = Emb_BMPEmbosserDriver         / use for Nippon BMP
#   Driver = Emb_BrailloEmbosserDriver     / use for Braillo
#   Driver = Emb_DotnPrint_EmbosserDriver  / use for ATC Dot n Print
#   Driver = Emb_EnablingEmbosserDriver    / use for Enabling
#    Software uses this as a key to know that escape codes may be used, ETCAllowEscapeCodes sets the use of escape
#    codes as a default
#   Driver = Emb_Gemini_EmbosserDriver     / use for Nippon Gemini
#   Driver = Emb_IndexEmbosserDriver       / use for Index
#   Driver = Emb_OhtsukiEmbosserDriver     / use for Ohtsuki
#   Driver = Emb_PumaEmbosserDriver        / use for Puma and Btec100
#   Driver = Emb_ThielEmbosserDriver       / use for some Thiel (not PortaThiel)
#   Driver = Emb_TigerEmbosserDriver       / use for Viewpoint Tiger
#   Driver = Emb_VersapointEmbosserDriver  / use for original VersaPoint
#   Driver = Emb_MountbattenEmbosserDriver / use for Mountbatten
#       Drivers come in two groups: normal and special. Normal drivers
#       behave in similar ways, differing in graphics and other specialized ways.
#       Special drivers offer very specialized support, and cannot be compared
#       Normal = Generic, Enabling, Index, Ohtsuki, VP, Puma, and Mountbatten
#       Specialized = Tiger, DotnPrint, BMP, Gemini, Elekul
# *** More information about Elekul 55 support in (dbt directory)\drivers\print55\dbt-help.txt
#
#   Encoding = display.cpt, British        / list the file name and the section
#                                          / name for different braille encodings
#   Encoding = display.cpt, French         / to be used for output. See the file
#                                          / display.cpt for more information
#   Encoding = display.cpt, German
#   Encoding = display.cpt, North American English
#   Encoding = display.cpt, Spanish
#   Encoding = pumavi.cpt,  PUMA binary (transparent) code
#
#   EndofPage = 0    / page ends with EOL (End of line, usually CR/LF), form feed (default)
#   EndofPage = 1    / no extra EOL before form feed at end of page *** new 10/15/2004
#   EndofPage = 2    / pad EOL (s)  to the end of the page, never use a form feed
#                    / This command is useful if you feel that a force new page is not working on
#                    / your embosser. It is vital that DBT knows the number of lines on each page,
#                    / or the top of form will drift high or low from one page to the next.
#   EndofPage = 3    / use a control K instead of a standard control-L (form feed)
#                    / This was needed for some embosser which are now museum pieces (i.e. very old)
#   EndofPage = 4    / nothing, but prompt 
#   EndofPage = 5    / use the Puma end of page character (hex 83).
#   EndofPage = 6    / use for ELEKUL 03 only:
#                    /   - pad EOL (s) to the end of the page, then write Control-R
#                    /   - also write Control-R at start of job
#
#   ETmulti              / multiple copies in the style of Enabling
#   ETCAllowEscapeCodes  /     Set the use of escape codes by the software as the default.
#                        / use 23.1E on the embosser to really turn on use of escape codes
#   ETMediumResGraphics  /     Model supports Enabling Technologies medium-res (esc-1) graphics
#   ETHighResGraphics    /     Model supports Enabling Technologies high-res (esc-6) graphics
#   ETSpeaks             /     Model supports Enabling Technologies 'ET Speaks' voice output
#   FormNamePrefix       / a string value set for some embossers to allow display of model-specific form names
#   BrailloHGapGraphics  / model supports a 'graphics' mode where line spacing is compressed
#                        / to produce an even pattern of dots *down* the page
#                        / (This is the same as RDC 'Gapped Graphics'.)
#   BrailloVGapGraphics  / model supports a 'graphics' mode where cell-to-cell spacing is
#                        / compressed on each line to produce an even pattern of dots *across* the page
#   LabellerROM          / embosser is designed primarily for labels (this is an optional ROM for some
#                        / Enabling embossers
#   LessLinesInterpoint = 1      / produces fewer lines if this
#                        / embosser is used as an interpoint embosser
#   Form 10_n =  85 x 100 ; SinglePage ; 34 x 22 ; 32 x 22
#                        / form definition, each embosser can have many.
#                        / This one says internal name is 10_n,
#                        / 8.5 by 10 inches in size (numbers are in tenths)
#                        / Pages are as individual pages
#                        / (not like Index 4x4 which are double sized)
#                        / maximum size for this paper form is 34 by 22,
#                        / though 32 by 22 is recommended
#                        / Please note the use of quotes, equal sign,
#                        / letter x, and semi-colon
#                        / Clipboarding is recommended to help avoid errors.
#                        / For Europeans, the setup process looks for forms in the following order:
#                                          A4, 12_w 11_w 11_n 10n
#                        / For all others, the setup process looks for forms in the following order:
#                                          11_w 11_n 10n
#   Handshaking          / Also called Flow Control, How DBT expects the embosser to issue a request to stop the flow
#                        / of characters from DBT. Choices are Hardware, Software, and None. Hardware is the default.
#                         / Hardware handshaking is also called DTR or RTS?CTS. Software handshaking is also called
#                        / Xon/Xoff. Virtually all serial interfaces with a PC use hardware handshaking. It is rare
#                        / to have a device that cannot do hardware handshaking and you need Software handshaking. 
#                        / None would only be useful to assist with debugging the handshaking.
#   HardwareDCB             / keyword to reprogram the defaults for what is called the DCB structure of the
#                        / serial port when the serial port is set for hardware handshaking)
#                        / See the Elekul 55 for a similar command. Also see the end of this file
#                        / For clues to keywords, do a Google search for: dcb structure msdn library
#   HelpPage             / Identifying which internal htm file is loaded when F1 is pressed at the list of embossers
#   Index16Graphics      / can do Index graphics, 1.6mm dimensions
#   Index20Graphics      / can do Index graphics  2.0mm dimensions
#                        / note: different ROM version of index use different placement of graphics size command
#                        / default is at 25th position. If you need another value, add Customize = 26 or whatever
#                        / putting in Customize = 1 suppresses the graphics resolution escape code for the model II Index
#                        / number works for your embosser
#   IndexMulti           / do multiple copies in the style of Index embossers
#   IndexPageInch        / Index embosser in which page size is in English units
#   IndexPagemm          / Index embosser in which page size is in metric units
#   IndexVersions        / Index embosser versions in which this model is available (1, 2, 3_early, 3, or 4)
#   IndexZfold           / this Index embosser supports Zfold embossing
#   IniFile              / Name of file in C:\Windows that controls Tiger interpoint
#   InitSequence         / Used for Index V3 (Firmware 10.25 or newer) and Index V4/V5 embossers only.
#                        / This defines the parameters, in order, to be sent in an Escape-D sequence at
#                        / the start of each job. Parameters may be listed with or without a value.
#                        / When a parameter is listed in InitSequence with a value, it will be sent as-is
#                        / (i.e. with the specified value) as part of the Escape-D sequence at the start
#                        / of each embossing job. If the parameter does not include a value, then in some
#                        / cases, DBT may add value, computed internally, based on the form definition.
#                        / For V4 and V5 embossers, DBT can add values to DP, CH, BI, LP, TM, and MC parameters.
#                        / For V3 embossers, DBT can add values to DP, PW, PL, IM, OM, TM, BM, and MC parameters.
#                        / If the parameter, as showing in InitSequence, does not contain a value and DBT
#                        / does not add one, it will be sent without one.
#                        / There is an exception for the MC parameter. This will never be sent without
#                        / a parameter and will never be sent with a parameter of 1. Neverthless, your
#                        / InitSequence should contain the MC parameter without a value, because this is
#                        / necessary for DBT to properly support multi-copy output.
#                        / DBT has two built-in default values for InitSequence: one for V3 and one for V4/V5.
#                        / If you define a blank InitSequence, the built-in default will be used. To eliminate
#                        / most DBT initialization of your embosser, but leave it possible for DBT to control
#                        / multi-copy output, simply set InitSequence to MC
#                        / DBT does little-to-no validation of InitSequence values, so take great care
#                        / and test thoroughly if you do modify it. Please refer to the relevant Index
#                        / Printer Interface Protocol manual, available from Index Braille, for more information
#                        / about Escape-D parameters and values.
#   Interline            / The embosser can do inkprint interlining
#                        / (the mechanics are done in the driver)
#   LettersUpperCase     / Send letters to the embosser in upper case only
#   LfUnix                 / only send line feed, not carriage return / line feed (not used)
#   LinesPadded          / pad lines with spaces
#   LineWrap             / if a line is full, do not send a carriage return
#   ManualInterpoint     / can do interpoint, but requires 2 passes (odd #'s, then even with prompt
#                          / in between -- see Nippon Gemini
#   MaximumCells = 40    / the absolute maximum cells that can be produced
#                        / assuming optimum conditions                                    
#   MaximumLines = 25    / the absolute maximum lines that can be produced
#                        / assuming optimum conditions
#   NarrowWarning        / a narrow page embosser only,
#                        / warn if the user wants wide paper or format
#   NoLeftMargin         / no embosser command to set the left margin
#   NoMakeupSides        / job needn't contain an even number of sides
#                        / (iff CanDoInterpoint is set, job is assumed to require an even number of sides)
#                        / (NoMakeupSides is not supported by some drivers)
#   NoParallel           / no parallel port in this embosser
#   NoSerial             / no serial port in this embosser
#   NoRightMargin        / no embosser command to set the right margin
#   NoTopMargin          / no embosser command to set the top margin
#   OneByOne             / output each character one at a time
#                        / (not line at a time), used for Ohtsuki
#   Parity = 1           / set default, 0 = no parity, 1= even, 2=odd, and 3= mark parity
#   PageWrap             / if a page is full, do not send a form feed; do not combine this with EndofPage = 1.
#   PlateLoadByte = n      / Setting of PED-30 data switch 2 (unused for other devices)
#                          / Note that, when this is set and output is directed to a serial port, setting
#                          / the Prompt for New Sheet setting will not display a prompt, but output will
#                          / be suspended until the specified byte is received.
#                          / If you have a serial cable that does not support carrying data back to the host,
#                          / then it may be necessary to remove the PlateLoadByte line from the PED-30
#                          / definition and rely on prompts displayed on the host to replace true handshaking.
#   PortCloseSyncByte  = n / Byte value to write, as padding, after output complete and before closing port.
#                          / A value of 0 is used if none is specified.  See also PortCloseSyncCount
#   PortCloseSyncCount = n / Number of bytes to write, as padding, after output complete and before closing port.
#                          / A value of 0 (no padding) is used if none is specified.  See also PortCloseSyncByte.
#                          / Very rarely should it be necessary to specify PortCloseSyncCount.
#   PortCloseSyncPrompt  / User will be prompted to 'Click OK when embossing complete' before port is closed.
#                        / This should rarely be necessary, but may help solve otherwise intractible problems
#                        / of data loss at the end of a job.
#   PromptNewPage        / each page must be inserted manually into the embosser
#   RotatePaperSizings	 / For Index 4x4, swap PL and PW dimensions 
#   SeparateGraphics     / Integer value: 0 - graphics may share page with text and other graphics
#                        / 1 - graphics must start a new page (text may only appear after)
#                        / 2 - graphics must end the page (text may only appear before)
#                        / 3 - graphics must start *and* end a page
#                        / Other values are undefined. 0 is implied when the keyword is absent
#   SoftwareDCB             / keyword to reprogram the defaults for what is called the DCB structure of the
#                        / serial port when the serial port is set for software (Xon/Xoff) handshaking.
#                        / This keyword is used in the Elekul 55. Also see the end of this file.
#                        / For clues to keywords, do a Google search for: dcb structure msdn library
#   SortToTop            / use to force an entry to be at the top of the list, ignoring alphabetical order
#                        / used for generic and generic interpoint
#   StartLeftMargin = 0  / this embosser calls the first cell position 0
#   StopBits = 2         / default for stop bits is 2 instead of 1
#   SubModel = n         / used to distinguish among different embosser models that use the same driver
#   SupportSeekTOF       / model supports seeking top-of-form (Enabling models only)
#   TranSend             / this embosser is compatible with TranSend SE (Enabling models only)
#   UsingEscapeCodes     / Escape codes are used; typical sequence will set page size to match document
#                        / This suppresses the prompt for embosser margins in embosser setup
#   VPGraphics           / do graphics in the style of the VersaPoint
#   VPmulti              / do multiple copies in the style of the VersaPoint
#   WindowsDriverName    / used to help DBT setup match plug and play information with embossers
#
#   YesFTDI              / this embosser has a USB port (for communications through FTDI)
#   YesUSB               / this embosser has a USB port (for raw communications)
#
#
#  Some Notes on Form Size
#
#  A common formula to determine the number of lines an embosser can do is (page length)/(0.4)
#  Thus the maximum for 11 inch paper is 27 lines, for 12 inch page is 30 lines
#  For Index Embossers, use  ((page length*25.4))/10    or (page length mm)/10  (round down, never up)
#  For Index embossers, use  ((carriage width)*(25.4))/6 - 6  or (carriage width mm)/6 - 6
#  For Index embossers there is sometimes a one line penalty for interpoint
#
#
# Please notice that some of these factors can be changed in the user
# interface, some cannot.
#

# comment. We are about to start real data.

#                              Handshaking
#
# To communicate on a COM port, one sets the COM port settings through a Windows structure called a DCB.
# There are many settings in a DCB; many of them are quite well explained in other documentation such as
# http://msdn.microsoft.com/library/default.asp?url=/library/en-us/devio/base/dcb_str.asp
# Also useful: http://en.wikipedia.org/wiki/Serial_port and http://en.wikipedia.org/wiki/RS-232
#
# We allow manipulation of the DCB outside of the DBT program defaults. The emb.elt file allows one to
# specify with precision what settings are written to the DCB for hardware handshaking and what settings
# are written to the DCB for software handshaking.  In fact, one can specify this on a device-by-device basis.
# See the entry for Elekul 55 for a sample of how this is done. This is repeated as a comment below for your
# convenience.


# Defaults for hardware handshaking by DBT are set as follows, all parameters are zero or false except for:
# fInX  = false
# FOutX = false
# fOutxDsrFlow = true
# fOutxCtsFlow = true 
# fDtrControl = DTR_CONTROL_HANDSHAKE;
# fRtsControl = RTS_CONTROL_HANDSHAKE;

# Defaults for software handshaking by DBT are set as follows, all parameters are zero or false except for:
# fInX  = true
# FOutX = true
# fOutxDsrFlow = false
# fOutxCtsFlow = false 
# fDtrControl = DTR_CONTROL_ENABLE
