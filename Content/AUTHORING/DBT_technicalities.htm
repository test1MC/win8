Duxbury Technicalities

First of all, the use of spaces in the middle of file names is very strongly deprecated. There are build-related scripts that collect lists of files and submit them for processing. A space in the middle of a file name can break the processing.

A few folders and files in the Flare project are constrained to retain an exact, specific name, or exact place in the folder hierarchy, unless companion changes are made in application code or in the autobuild process. Those constraints are listed here.

Folder: activation_wizard_help

First, to keep the context-sensitive Help working in the activation wizard (i.e., direct page access from each wizard page to its Help topic), all the wizard help files must remain in the “activation_wizard_help” folder, and this folder must remain directly under the “Contents” folder.

Second, we cannot change this folder name or the names of the help file topics without changing the activation wizard code.

Skin File for DBTHelp: “DuxburyMainChm”

The program code in DBT/Winthat opens DBTHelp (helpfile.chm) also checks that the name of the window it opens matches a name that it has internally hard-coded: "DuxburyMainChm".

The Flare file that supplies that window name is a skin file named "DuxburyMainChm.flskn." This file name must not be changed, nor a different skin file chosen for the DBTHelp target(s), or DBTwill throw a spurious "not found" error when it opens the Help file.

If we change the skin name, we have to change the name in the code, and vice versa.
